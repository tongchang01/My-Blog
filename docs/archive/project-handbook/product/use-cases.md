# V2 用例清单

> 本文档回答："V2 中不同角色能做什么？哪些动作明确不能做？"
> 范围：基于 `feature-inventory.md` ⑳ 项决策与 `decisions-draft.md` R1-R8，不新增功能范围。

## 1. 角色

| 角色    | 含义                          |
| ----- | --------------------------- |
| GUEST | 未登录访客。可阅读公开内容、提交评论、访问留言板和友链 |
| DEMO  | 演示账号。可登录后台查看内容，但不得执行写操作     |
| ADMIN | 站长本人。拥有全部后台写权限              |
| 系统任务  | 定时发布、统计聚合、日志清理等后台自动动作       |

## 2. GUEST 用例

### G-01 选择语言并浏览站点

- 访客第一次进入站点时，系统按浏览器语言选择 `zh / ja / en` 中的默认语言。
- 访客可以手动切换语言，后续访问使用上次选择。
- URL 必须带语言前缀，如 `/zh/posts`、`/ja/posts`、`/en/posts`。
- UI 文案随语言变化；文章正文保持中文，标题和摘要按当前语言展示，缺失时 fallback 到中文。

### G-02 浏览文章列表

- 访客可以查看 PUBLISHED 文章列表。
- 访客不能在公开列表看到 DRAFT、PRIVATE、未到发布时间的 SCHEDULED 文章。
- PASSWORD 文章可以出现在列表，但只显示标题、摘要、锁标识和公开元数据，不展示正文和评论内容。
- 列表支持按分类、标签、归档月份筛选。

### G-03 查看文章详情

- 访客可以直接查看 PUBLISHED 文章正文。
- PASSWORD 文章详情必须先通过文章密码校验，通过后才能查看正文、评论列表和评论入口。
- PRIVATE、DRAFT、未到发布时间的 SCHEDULED 文章对访客不可见。
- 文章详情展示分类、标签、正文、评论入口和基础 SEO 所需的标题、摘要、canonical 信息。

> 目标能力：PASSWORD 完整解锁链路位于上线后增量。首版公开详情固定返回 `403 + 10003`，前端只实现锁定占位态，不调用 `/unlock`。

### G-04 搜索公开文章

- 访客可以搜索公开文章标题和摘要。
- 搜索结果只包含访客有权查看的文章。
- 搜索接口受限流保护，防止爬虫高频请求。

### G-05 查看分类、标签、归档

- 访客可以查看分类列表、标签列表和归档页。
- 分类与标签名称按当前语言展示，缺失时 fallback 到中文。
- 分类和标签只统计未删除且可公开展示的文章。

### G-06 发表评论

- 访客可以对允许评论的 PUBLISHED 文章发表评论。
- PASSWORD 文章必须先通过文章密码校验，才能读取评论或发表评论。
- 评论作者必须填写昵称和邮箱，个人站点可选。
- 评论内容以 Markdown 子集提交，系统保存原文和清洗后的安全 HTML。
- 未命中黑名单的评论立即公开；命中黑名单的评论进入待审，不公开展示。

> 首版范围：PASSWORD 评论列表和评论提交固定返回 `403 + 10003`，解锁后的评论能力随完整解锁链路后置。

### G-07 回复评论

- 访客可以回复已公开的评论。
- 回复最多两层结构：回复子评论时仍挂到同一个顶层评论下。
- 不允许回复已隐藏、待审或已删除的评论。
- 不允许跨文章或跨留言板目标回复。
- 被回复者可收到邮件通知。

### G-08 使用留言板

- 访客可以在留言板发表留言和回复。
- 留言板复用评论能力，但目标固定为 GUESTBOOK。
- 留言板不显示文章上下文，不计算文章评论数。

### G-09 查看友链

- 访客可以查看状态为显示的友链。
- 友链说明为单中文字段，不做三语副本。
- 友链申请不在 V2 表结构内实现；访客通过邮件或 GitHub issue 联系站长。

### G-10 查看关于页和站点配置

- 访客可以查看关于页、站点标题、副标题、Logo、favicon、公开社交链接和 Spotify Embed。
- 关于页内容按当前语言展示，缺失时 fallback 到中文。
- 活动热图展示公开统计结果，不暴露访客个人信息。

### G-11 使用 Sitemap / Robots

- 搜索引擎可以读取 Sitemap 和 robots.txt。
- Sitemap 只包含允许收录的公开页面。
- PASSWORD 文章入口页可收录，但不得输出正文。

## 3. DEMO 用例

### D-01 登录后台演示账号

- DEMO 可以使用后台登录入口登录。
- DEMO 登录后获得只读后台视图。
- DEMO 的 token 策略与 ADMIN 相同：access token 短期有效，refresh token 可刷新。

### D-02 查看后台数据

- DEMO 可以查看仪表盘、文章列表、分类、标签、评论、友链、附件和站点配置。
- DEMO 可以查看评论待审列表和统计图表。
- DEMO 只能查看 PUBLISHED 正文；DRAFT、PRIVATE、PASSWORD、SCHEDULED 的后台详情正文固定为 `null`。
- DEMO 可以查看评论管理列表，但邮箱、IP、User-Agent 三个审计字段固定为 `null`。
- DEMO 不能新增、修改、删除、审核、发布或上传任何资源。

### D-03 退出登录

- DEMO 可以主动退出登录。
- 退出后 refresh token 失效，后续必须重新登录。

## 4. ADMIN 用例

### A-01 登录、刷新会话和退出

- ADMIN 可以登录后台。
- 登录成功后获得 access token 和 refresh token。
- access token 过期后可用 refresh token 换取新的 token 对。
- ADMIN 退出登录后当前 refresh token 失效。
- ADMIN 修改密码后，历史 access token 和 refresh token 全部失效。

### A-02 管理文章

- ADMIN 可以创建、编辑、删除和恢复文章。
- ADMIN 可以设置文章状态：DRAFT、PUBLISHED、PRIVATE、PASSWORD、SCHEDULED。
- ADMIN 可以配置文章标题和摘要三语副本，正文保持中文 Markdown。
- ADMIN 可以设置分类、标签、封面、slug 和发布状态；逐篇关闭评论不进入 V2 起点 schema。
- ADMIN 可以导出单篇文章 Markdown。
- ADMIN 不需要批量导入文章。

### A-03 管理分类和标签

- ADMIN 可以新增、编辑、删除和排序分类。
- ADMIN 可以新增、编辑和删除标签。
- 分类和标签名称支持三语副本，slug 必须唯一。
- 已被文章引用的分类或标签删除前必须由应用层给出明确处理方式。

### A-04 审核和管理评论

- ADMIN 可以查看 PASS、PENDING、HIDDEN 三类评论。
- ADMIN 可以将 PENDING 改为 PASS 或 HIDDEN。
- ADMIN 可以隐藏、恢复或软删除评论。
- ADMIN 可以查看评论作者邮箱、IP 和 UA，用于审计和防刷。
- ADMIN 不能直接渲染或信任评论原始 Markdown。

### A-05 管理友链

- ADMIN 可以新增、编辑、隐藏、显示和删除友链。
- 友链没有在线申请审核状态；申请流程在站外完成。

### A-06 管理站点配置

- ADMIN 可以编辑站点标题、副标题、关于页、Logo、favicon、备案号和 Spotify 播放列表 ID。
- 展示型站点配置支持三语副本；非展示型配置保持单值。

### A-07 管理附件

- ADMIN 可以上传、查看和删除附件。
- 附件上传必须校验 MIME、大小和 SHA-256。
- 命中相同 SHA-256 的附件时，系统复用或恢复已有记录，避免重复存储。

### A-08 查看仪表盘和统计

- ADMIN 可以查看总文章数、总评论数、本月发布数、最新评论、热门文章 TOP 10、访问趋势和语言分布。
- 原始访问明细只保留短期；长期展示依赖日聚合数据。

### A-09 管理个人资料

- ADMIN 可以编辑昵称、头像、个人简介、公开邮箱、所在地、个人主页和社交链接。
- ADMIN 可以修改密码。

## 5. 系统任务用例

### S-01 定时发布文章

- 系统按 Asia/Tokyo 时间检查 SCHEDULED 文章。
- 到达发布时间后，系统将文章转为 PUBLISHED。
- 定时任务更新产生的 `updated_by` 为空。

### S-02 访问统计聚合

- 系统把页面访问明细聚合到日统计。
- 聚合口径按文章、语言和 JST 日期计算。

### S-03 清理短期数据

- 系统定期清理过期 refresh token、访问明细和邮件失败日志。
- 清理任务只物理删除明确属于短期保留的数据，不物理删除业务实体。

### S-04 发送评论通知邮件

- 评论回复触发邮件通知。
- 邮件发送失败经过重试后仍失败时，系统记录失败日志。
- 邮件日志不记录敏感 token，不暴露内部异常细节。

## 6. 明确不做

- 不做访客注册、QQ OAuth、邮箱验证码登录、在线用户管理和强制下线。
- 不做动态后台菜单、完整 RBAC、操作日志入库、异常日志入库。
- 不做说说、相册、自建音乐库、Quartz 后台管理。
- 不做自建 Newsletter 系统。
- 不在 V2 schema 内预留 Umami / Plausible 抽象层。
