# V2 第一版发布范围复评

> 状态：当前评估
> 适用范围：MyBlog V2 第一版发布前范围裁剪
> 日期：2026-07-07
> 对应代码：`frontend/apps/blog/`、`MyBlog-springboot-v2/`

## 结论

第一版目标收敛为：尽快打通个人博客主流程体验，而不是复刻旧 Aurora 全部功能，也不是提前经营 SEO / RSS / 社交传播能力。

主流程定义为：

- 访客能稳定进入首页。
- 能浏览文章列表、置顶/推荐、文章详情。
- 能按分类、标签、归档和搜索找到文章。
- 能查看关于页。
- 能查看文章评论并提交文章评论。
- 页脚能展示基础访问统计和建站天数。
- 后台能完成内容生产、分类标签、评论审核、附件、站点配置和统计查看。
- 生产部署相关的环境变量、反向代理、客户端 IP、S3、备份恢复和上线冒烟可验证。

## 当前前台缺口

### 第一版前建议处理

1. **友链页简版**
   - 现状：`frontend/apps/blog/src/pages/links.vue` 仍通过旧 `articleStore.fetchArticle('links')` 读取 `/api/pages/links/index.json` 形态数据，并挂旧第三方评论插件。
   - 后端已有 `GET /api/public/friend-links`。
   - 建议：第一版只接公开友链列表，展示简单卡片；不复刻头像墙、分组、随机访问、申请说明、友链评论和页面统计。

2. **部署硬项**
   - 生产环境变量、反向代理、CORS、可信代理 / 客户端 IP、S3、备份恢复、上线冒烟仍未实战验证。
   - 这些比 SEO、留言板、PASSWORD 完整解锁更接近上线风险。

### 第一版后置

1. **O-001 PASSWORD 文章完整解锁**
   - 首版保留锁定态即可。
   - 完整 unlock API、Article Access Token、解锁态前端和 PASSWORD 评论链路后置。

2. **O-019 留言板评论**
   - 文章评论已接入 V2。
   - 留言板评论迁移后置，不阻塞第一版阅读主流程。

3. **完整 SEO / 公开索引增强**
   - 个人网站当前不经营流量，完整 SEO、RSS、Sitemap、Open Graph、结构化数据收益低。
   - 第一版不做重型 SEO。
   - 后续如果希望公开经营、提高收录或分享质量，再单独规划。

4. **Spotify Embed**
   - 与主流程弱关联，后置。

5. **Markdown chunk 分包、Sass `@import` / legacy API 清理**
   - 属构建和维护质量优化，不阻塞第一版。

6. **第三方评论工具物理删除、RecentComment 侧栏重建**
   - 文章评论主链路已不依赖第三方插件。
   - 留言板、旧动态 page 和 RecentComment 仍可能引用旧插件代码；等留言板策略确定后再清理，避免误删旧消费者。

## SEO 裁决

第一版不做完整 SEO 能力。发布清单中 SEO 与公开索引调整为后置增强项，仅保留生产暴露范围检查：

- 后台、OpenAPI、Swagger UI 不应被公开索引。
- PASSWORD 文章不得通过 meta、RSS、Sitemap 等扩展字段泄露正文。
- 如果后续启用 sitemap / RSS / Open Graph，再按公开 URL 策略补完整规则。

## 下一步建议

1. 先实现友链简版，关闭或下调 O-003。
2. 进入部署硬项文档和实战校准。
3. 手动部署跑通后，再决定是否设计 CD。
4. 第一版发布后再排 O-001、O-019 第二批、SEO 增强和旧插件清理。
