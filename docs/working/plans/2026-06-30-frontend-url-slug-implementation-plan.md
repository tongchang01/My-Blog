# 前台公开 URL 与 slug 生命周期实施思路

> 状态：实施思路
> 适用范围：前台 blog、后端 content、公开 API、后台文章/分类/标签编辑
> 最后校准：2026-06-30
> 权威程度：过程计划，裁决后需提炼到 handbook

## 本文档回答什么问题

本文档记录已经裁决的公开 URL 标识策略与 slug 生命周期处理方向，作为后续拆分正式实现任务前的实施依据。

本文档不直接替代 `docs/handbook/start-here/open-issues.md`、`docs/handbook/api/` 或业务规则文档。实现完成后，需要把最终结论提炼回 handbook。

## 已定方向

### 1. 公开 URL 采用混合策略

前台公开读者端 URL 不强行统一为全 slug 主导。文章继续 ID 主导，分类和标签使用 slug 主导。

目标形态：

```text
/:lang/posts/:id/:slug?
/:lang/categories/:slug
/:lang/tags/:slug
```

说明：

- 文章公开 URL 以 ID 定位，slug 只增强可读性，继续遵循 BR-203。
- 分类和标签公开 URL 以 slug 定位。
- 后端内部仍可以继续使用 ID 作为 Entity / Domain / Application 的主标识。
- 后台 admin 的内部管理接口可以继续使用 ID。
- 分类/标签公开筛选可以由后端支持 slug 查询，或由前台集中解析 slug 后复用 ID 查询；实现时保持转换集中，避免散落在组件中。

### 2. slug 生命周期以“文章宽松、分类标签稳定”为原则

文章 URL 有 ID 兜底，slug 修改不应导致文章链接失效。分类和标签没有 ID 兜底公开路径，slug 一旦进入公开 URL，就具有长期链接含义。

当前不做 slug history / redirect / alias 表，先通过锁定规则控制生命周期复杂度。

### 3. 文章 slug 规则

- 文章继续遵循 BR-203：公开路径以 ID 定位，slug 只增强可读性。
- 文章 slug 可为空、可修改，不作为唯一业务标识。
- 文章详情路由继续使用 `/:lang/posts/:id/:slug?`。
- 文章 API 继续以 ID 查询为主，不因本轮分类/标签 slug 接入改成 slug 主导。
- 如未来单独决定文章 slug 主导，再新增 ADR、迁移策略和兼容方案。

### 4. 分类 / 标签 slug 规则

- 分类和标签创建后 slug 锁定。
- 名称可以继续修改，slug 不随名称自动变化。
- 后台编辑分类/标签页面应把 slug 展示为只读字段。
- 新建分类/标签时必须显示明确提示：slug 创建后将作为公开 URL 的一部分，后续不建议修改。
- 当前后端已经要求分类/标签 slug 唯一，且软删除后不可复用；该规则继续保留。

### 5. 后台提示要求

分类新建、标签新建页面必须增加对 slug 重要性的提示。文章页面可保留 slug 可读性提示，但不需要按公开后锁定处理。

提示内容应覆盖：

- slug 会出现在公开 URL 中。
- 分类/标签创建后可能无法在普通编辑页面修改。
- 修改 slug 会影响旧链接。
- 建议使用简短、稳定、英文小写、数字和短横线组成的 slug。

示例文案：

```text
slug 会作为公开 URL 的一部分。分类和标签请尽量一次性确定，创建后将锁定；后续修改可能导致旧链接失效。
```

## 影响面

### 后端 content

需要评估和修改：

- 文章 slug 继续作为可选可改的可读性增强，不升级为公开唯一标识。
- 文章详情公开查询继续按 ID 查询。
- 分类/标签更新服务禁止修改 slug。
- 分类/标签公开文章查询支持 slug。
- 锁定后如果收到 slug 修改请求，应返回明确业务错误。

### 前台 blog

需要评估和修改：

- 文章详情路由保持 ID 主导。
- 分类页和标签页使用 slug 路由。
- 搜索、归档、文章卡片继续生成文章 ID 主导链接，侧栏分类/标签点击生成 slug URL。

### 后台 admin

需要评估和修改：

- 文章编辑页不需要因公开状态锁定 slug。
- 分类/标签编辑页 slug 只读。
- 新建页增加 slug 重要性提示。
- 若用户修改标题/名称，不自动改 slug。

### API 文档和业务规则

需要同步更新：

- `docs/handbook/api/article.md`
- `docs/handbook/api/category-tag.md`
- `docs/handbook/product/business-rules.md`
- 必要时新增 ADR，说明公开 URL 标识策略。

## 待确认点

以下问题尚未最终裁决，实施前需要确认：

1. 分类/标签公开筛选由后端直接支持 slug 参数，还是前台集中解析 slug 到 ID 后复用现有 article 查询。
2. 分类/标签现有数据如果有空 slug 或重复 slug，实施前如何清理。
3. 分类/标签 slug 锁定后是否保留管理员维护工具，本轮默认不做。
