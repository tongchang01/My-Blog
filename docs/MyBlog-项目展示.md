# MyBlog：一个前后端分离的个人博客系统

很多时候，博客不只是一个写文章的地方，更像是一个持续生长的个人产品。

这套 `MyBlog` 就是我自己在使用和持续迭代的博客系统。它不仅承担了文章发布、内容管理、评论互动这些基础能力，也逐渐加入了相册、说说、定时任务、音乐播放器、权限管理等更完整的网站能力。

这篇文章不打算只讲“我用了什么框架”，而是想更完整地介绍一下这个项目的结构、功能设计，以及我在这个过程中做过的一些技术取舍。

---

## 一、项目定位

`MyBlog` 是一个完整的前后端分离博客项目，分为三部分：

- 前台博客：负责内容展示、文章阅读、评论互动、相册和说说等页面
- 后台管理端：负责文章、分类、标签、评论、用户、权限、网站配置等管理操作
- 后端服务：负责业务接口、权限控制、数据存储、定时任务和第三方服务对接

相比“单纯的博客模板”，我更希望它是一套可以长期维护、持续扩展的内容平台。

---

## 二、整体技术架构

### 前端部分

前端我拆成了两个项目：

- `MyBlog-blog`：前台博客，基于 `Vue 3 + TypeScript + Pinia + Element Plus`
- `MyBlog-admin`：后台管理端，基于 `Vue 2 + Vuex + Element UI`

前台更偏重页面体验、交互和展示效果，后台则更强调稳定的管理能力和较成熟的组件生态。

### 后端部分

后端基于 `Spring Boot` 构建，主要使用了：

- Spring Boot 2
- Spring Security
- MyBatis-Plus
- MySQL
- Redis
- RabbitMQ
- Quartz
- Knife4j

此外，项目里还接入了：

- AWS S3：用于对象存储
- AWS SES：用于邮件发送

## 四、我做过的一些定制功能

这个项目对我来说比较有意思的部分，不是把基础博客跑起来，而是持续往里面加自己真正会用到的能力。

### 1. 音乐播放器模块

最近我在这套博客里补上了音乐播放器相关的能力，包括：

- 前台集成播放器组件
- 后台增加音乐配置和曲库管理
- 后端新增音乐相关接口
- 数据库补充音乐表和权限资源

播放器现在的交互逻辑也做过一次调整：

- 去掉了“右下角固定”和“悬浮停靠”的切换配置
- 最终只保留“悬浮停靠”这一种交互方式
- 并且把播放器位置上移，避免遮挡页面右下角的其他交互区域

这类功能虽然不算核心业务，但很能体现一个项目是否在被真实使用、真实打磨。

### 2. 网站配置能力

后台的网站配置不只是一些文本字段，而是把站点里很多可见的内容都做成了可配置项，比如：

- 网站名称
- 作者信息
- 头像和 logo
- 网站公告
- 社交链接
- 评论审核
- 邮件通知
- 打赏开关
- 音乐播放器配置

这能让我在不改代码的前提下，直接通过后台调整站点展示内容。

### 3. 定时任务与日志

项目里还集成了 Quartz，并配套做了：

- 定时任务管理
- 任务执行日志
- 操作日志
- 异常日志

这些能力放在一个个人博客里可能不算“必须”，但它会让整个系统更接近一个长期可维护的后台产品，而不只是页面集合。

## 五、为什么我要做成三端结构

如果只是为了写文章，做一个单体博客其实更快。

但我最后还是把它拆成了：

- 前台
- 后台
- 后端

原因很简单：

### 1. 展示层和管理层天然不同

前台强调的是阅读体验、页面风格、访问性能。

后台强调的是配置能力、可操作性、权限控制和管理效率。

把这两者拆开之后，很多设计决策会清晰很多。

### 2. 便于后续扩展

拆成三端之后，后续无论是继续补相册、音乐、说说，还是加新的管理模块，都不会把前台展示逻辑和后台管理逻辑揉在一起。

### 3. 更贴近真实项目结构

对我自己来说，这个项目除了“自己在用”，也承担了一部分练习完整项目设计和工程组织的作用。

它不是一个 demo，而是一套更贴近真实业务结构的小型系统。

## 六、我比较看重的几个技术点

### 1. 前后端分离的边界清晰

两个前端项目都通过 `/api` 代理到后端，职责划分比较明确：

- 前台负责展示
- 后台负责管理
- 后端负责业务和数据

### 2. 功能可以逐步迭代，而不是一次性写死

像音乐播放器、网站配置、菜单权限这些能力，都是在原有系统之上逐步加进去的。

这也意味着，项目结构本身要允许后续继续长出新功能。

### 3. 后台不是“为了有后台”

很多个人项目会做一个很薄的后台，只够改文章。

但我更想让后台真正具备内容运营的意义，所以才继续加上：

- 权限资源
- 菜单控制
- 用户管理
- 在线状态
- 日志系统
- 定时任务

这些模块让项目更像一个完整系统，而不是单页工具。

## 七、如果把它当成作品集项目，它能体现什么

如果从作品集角度看，这个项目比较能体现的，不是某个单点技术，而是下面这些能力：

- 能独立搭起前台、后台、后端三部分结构
- 能把展示站点和管理系统分开设计
- 能从“博客”逐步扩展到“内容平台”
- 能处理配置化、权限、日志、定时任务这类更偏工程化的问题
- 能基于真实使用场景持续做增量迭代

它未必是那种“最炫”的项目，但它是一个足够完整、足够连续演进的项目。

- 更完整的部署文档
- 更清晰的环境变量管理方式
- 更好的后台可视化能力
- 更细粒度的站点配置项
- 更完善的媒体资源管理能力

对我来说，这个博客系统本身就是一个长期迭代的技术作品。

## 九、结语

很多项目在完成“可运行”之后就结束了，但我更喜欢那种能持续迭代、持续贴近真实使用场景的项目。

`MyBlog` 对我来说就是这样一个项目。

它既是我的博客，也是我的后台系统、我的工程练习场，以及一个会继续成长的个人产品。

如果你也在做自己的博客系统，我其实很推荐把它当成一个真正的项目去经营。因为博客最有意思的地方，不只是“写内容”，而是它会逼着你去面对一个完整网站真正需要的那些技术问题。
