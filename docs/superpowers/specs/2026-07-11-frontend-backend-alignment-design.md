# 前后台数据对齐设计

## 目标

在部署 V2 前，清理博客端仍依赖主题默认值或未处理失败状态的页面，让已有公开接口真正驱动前台展示；不新增后端接口，不改变现有 API 契约。

## 已确认问题

1. `/api/public/site-config` 已返回站点标题、作者头像等配置，但映射层没有把 `siteTitle` 写入主题品牌字段，页头和页脚仍可能显示默认值。
2. `/api/public/author-profile` 已返回作者昵称、头像、多语言简介及社交链接，作者 store 只更新局部视图模型，没有同步页头/页脚使用的主题站点字段；桌面侧栏和移动菜单重复触发请求。
3. 友情链接页面已经接入接口，但请求失败时没有错误状态，用户只能看到空白或骨架屏。
4. 后端已有 `/api/public/guestbook/comments`，博客端尚未提供留言板页面；该能力属于现有接口的前端缺口，后续按现有评论组件复用实现。
5. 文章密码解锁接口当前并不存在，保持现状，不通过猜测新增接口。

## 实施边界

- 复用现有 API、DTO、评论模型和主题模型。
- 先修复站点配置与作者资料的已有字段映射，再补错误状态和留言板接入。
- 每个改动拆成独立提交并运行对应前端测试；完成后再运行博客端完整检查。
