# 前台 Blog 上线后补充项

> 状态：当前有效
> 适用范围：V2 前台读者端
> 最后校准：2026-07-09
> 对应代码：`frontend/apps/blog/`
> 权威程度：前台体验延后项记录

## 本文档回答什么问题

本文档只记录“已确认不阻塞第一版发布，但上线后值得继续补齐”的前台读者端事项。第一版发布前仍以主流程可用为目标：公开文章浏览、文章详情、分类/标签/归档、搜索、关于、友链、文章评论、访问统计和基础作者卡片优先。

## 作者卡片

- 作者卡片社交链接已改为读取后端公开作者资料接口 `GET /api/public/author-profile`。
- 第一版只映射现有 UI 能展示的平台：GitHub、Twitter、知乎、掘金。
- 后端资料中已有但前台暂未展示的平台：LinkedIn、Qiita、个人主页、公开邮箱。原因是当前 `Social.vue` 和 `Social` 模型没有对应入口，强行接入会扩大 UI 和图标改造范围。
- 作者卡片底部“文字”总字数暂不修。当前前端仍显示 `0`，后续应改为由后端统计文章总字数，或在前台统一统计已公开文章正文后缓存；不要在本轮为单个卡片临时扫描所有文章。

## 文章详情

- “上一篇 / 下一篇”暂未恢复。当前公开文章详情 API 没有返回相邻文章信息，后续建议由后端按发布时间和 ID 返回上一篇、下一篇，前台只负责渲染。
- 文章顶部阅读信息目前以前台从正文计算字数和阅读时间为准。若后续需要跨页面一致，建议把字数、预计阅读时间沉淀为后端文章派生字段或查询结果字段。
- 目录、右侧作者卡、右侧固定栏已作为第一版体验的一部分恢复；后续只做细节优化，不再回退到旧 mock 数据。

## 公开索引和分享

- 完整 SEO、RSS、Sitemap、Open Graph、结构化数据继续后置。
- 当前个人站不以流量经营为目标，SEO 不作为第一版发布阻塞项；上线后如果需要更好的搜索引擎收录和社交分享预览，再按部署域名、文章 canonical URL、封面图和摘要策略统一设计。

## 留言板和评论扩展

- 留言板评论接入继续后置。
- 文章评论主链路已接入 V2 自研 API，但旧第三方评论和通用 page 物理清理需要单独分支做，避免和读者端体验修补混在一起。

## 后台联动

- 后台筛选组件扩展暂不阻塞第一版前台发布。
- 与前台强相关的后台字段维护优先级更高，例如作者资料的社交链接、头像、简介；纯管理效率优化可放到上线后。
