# 管理端后端能力消费审查

> 状态：当前有效
> 适用范围：`frontend/apps/admin/` 在**不修改后端接口、领域规则或样式体系**前提下的交互补齐
> 最后校准：2026-07-15
> 对应代码：`frontend/apps/admin/src/`、`MyBlog-springboot-v2/src/main/java/`
> 权威程度：管理端能力对齐与后续拆分的事实清单

## 1. 结论与边界

本次逐一比对了管理端 API 调用、页面入口和后端所有管理端控制器。结论如下：

1. 管理端的文章、分类、标签、评论、友链、附件、站点配置、账号资料、密码和统计写接口均已有前端入口；不存在需要为了“补齐”而新建管理端后端接口的缺口。
2. 截至当前分支，文章列表的分类/标签/时间范围筛选、统计看板日期区间和友链无效筛选均已完成前端对齐；后续重点转为表单语义、操作反馈和视觉改造前的模板清理。
3. 多个页面已经调用接口，却没有解释服务端状态机、全量写入、软删除、公开可见性和统计口径；这些是前端交互与表单语义缺口，不是样式任务。
4. 友链历史上的关键字和状态筛选曾是反向不一致；当前前端仅发送后端支持的 `page`、`size`。后续不得重新引入当前页本地过滤来冒充服务端分页筛选。

本清单只安排前端变更：复用现有 API、`use*` 管理函数、Element Plus 控件和现有 i18n 结构；不新增接口、不改变返回字段、不修改后端业务规则，也不包含管理端视觉重做。

## 2. 审查方法与判定标准

### 2.1 事实来源

- 后端路由与参数：各模块 `Admin*Controller`、`CurrentUserController`、`AuthController`。
- 服务端规则：对应 application service、domain model 与 request support；不能只依据 OpenAPI 注释推断。
- 前端实际消费：`frontend/apps/admin/src/api/`、`features/` 内的调用点与页面入口。仅存在于 API 封装或测试、没有页面调用的函数视为“未消费”。

### 2.2 标记

| 标记 | 含义 | 后续处理 |
| --- | --- | --- |
| 已消费 | 页面已按后端契约使用 | 仅补充必要的业务提示 |
| 部分消费 | 已调用同一接口，但遗漏后端已有字段或查询参数 | 只改前端模型、查询构造和控件 |
| 未消费但无需入口 | 详情与列表字段重复，新增页面不会带来用户价值 | 保留 API 封装，不为覆盖率强行加入口 |
| 反向不一致 | 前端声明/发送的能力不被后端支持 | 不做伪筛选；先删除错误预期或另开后端议题 |

## 4. 已完成的查询对齐与后续约束

以下任务均已在当前分支完成。保留其事实与约束，避免后续视觉改造回退查询语义。

### P0：已修正真实的查询语义

#### A-01 文章列表：已消费全部现有筛选参数

**完成证据**：`ArticleListFilters`、`buildArticleListParams` 和列表页已包含分类、标签、创建时间范围、发布时间范围；页面复用初始化时加载的分类与标签，并在提交前阻止反向时间范围。

**当前行为**：

- 筛选卡提供分类、标签、创建时间范围、发布时间范围；继续复用已加载的 `categories`、`tags`。
- 可选筛选仅在用户填入时进入请求；重置会清空全部字段并回到第 1 页，分页和刷新保留当前筛选。
- 时间使用既有 JST 显示约定并提交 ISO `LocalDateTime`；反向时间范围会在请求前阻止。

**不可做**：后端没有文章列表排序参数，不能增加“按评论数/更新时间排序”控件或在当前页本地排序后宣称它是查询结果。

**验证**：`query.ts` 和文章列表 hook 测试覆盖参数构造、筛选传递、重置和反向时间范围。

#### A-02 统计看板：已暴露日期区间

**完成证据**：仪表盘已提供自定义日期范围、近 7 天/近 30 天快捷项和重置；`useStatsDashboard` 会在请求前校验双端、顺序和最长 366 天。

**当前行为与仍需保留的语义**：

- 页面允许自定义范围和“近 7 天 / 近 30 天”快捷项；缺省请求遵循后端最近 30 个 JST 自然日。
- 自定义范围会校验“同时填写、开始不晚于结束、最多 366 天”。
- `todayPv`、`todayUv` 始终表示 JST 当天，即使查询范围为历史区间；顶部文章 `dailyUvSum` 是区间内逐日 UV 之和而非跨日去重访客数。视觉改造不得弱化这些说明。

**验证**：`query.ts`、hook 与页面测试覆盖 JST 快捷范围、合法/非法日期区间及筛选交互。

#### A-03 友链：已停止显示无效的服务端筛选预期

**完成证据**：`FriendLinkListFilters` 与管理 hook 现仅保留 `page`、`size`，页面已移除关键字与状态筛选控件。

**当前行为**：

- 不发送后端不支持的关键字/状态参数，不使用当前页本地过滤伪装为服务端筛选。
- 如产品确实需要全局友链筛选，另开后端任务，为查询契约、索引、分页语义和测试单独设计；不混入本轮。

**验证**：友链 API 与管理 hook 测试确认列表请求只含 `page`、`size`，页面不再出现无效筛选控件。

## 5. P1：让已有业务规则在操作前可理解

### B-01 文章编辑器：状态驱动的字段语义

| 规则 | 后端事实 | 当前后台 | 前端应明确的语义 |
| --- | --- | --- | --- |
| 完整 PUT/POST | 15 个文章字段必须全部出现在请求中；可选字段要显式传 `null` | 当前 payload 正确 | 保持统一 payload；不要把“可留空”误解为可以省略字段 |
| 草稿 | 后端允许草稿缺少中文标题、正文和分类 | 当前前端无论状态都强制中文标题、中文摘要、正文 | 校验按状态放宽：草稿不要求标题/正文/分类；摘要在任何状态都不是后端必填。若产品决定草稿也必须完整，应另行收紧后端，而非悄悄由前端单独限制 |
| 非草稿 | `PUBLISHED`、`PRIVATE`、`PASSWORD`、`SCHEDULED` 都需要中文标题、正文、分类 | 仅分类条件已部分实现 | 在状态切换后即时展示必填说明；不要等提交后才显示错误 |
| 发布时间 | `SCHEDULED` 必填且不得早于当前时间；`PUBLISHED`、`PASSWORD` 创建时为空则后端写当前时间；更新已有文章时后端保留原发布时间 | 仅 `SCHEDULED` 显示时间选择，且只校验非空 | 标明 JST、补“定时必须是未来时间”；不要承诺编辑已发布文章能改发布时间 |
| 密码文章 | 创建 `PASSWORD` 必须提供密码；编辑时传 `null` 会保留已有哈希；非密码状态不得携带密码 | 已有“留空保留”提示 | 将“仅用于 PASSWORD”与“状态切走后不会保留密码”写明；前端不展示、更不能回显旧密码 |
| 首页槽位 | 仅 `PUBLISHED` 可设置；`PINNED` 最多 1 篇、`FEATURED` 最多 2 篇 | 已禁用非发布状态的槽位 | 展示配额和冲突后果；不在前端猜测剩余名额，保存时如收到冲突应保留表单并显示可行动错误 |
| 标签 | 最多 20 个，不得重复，引用必须为活动标签 | 多选去重后提交，但没有上限提示 | 加选择上限与计数；保持重复 ID 不进入 payload |
| slug | 可空；非空时会 trim、小写化，格式为小写字母/数字/连字符，最长 160 | 只有示例占位符 | 说明“可留空、非唯一身份”；在输入时校验格式并提示会转小写 |
| 封面 | 非空时必须引用活动图片附件 | 选择器来自活动附件 | 提示删除已选附件会使后续保存失败；不接受手填非附件 ID |

### B-02 评论：目标、权限与计数影响可见

- 留言板评论的固定目标为 `GUESTBOOK / 0`；当筛选类型选为留言板时，应隐藏并清空“目标 ID”，列表中显示“留言板”而非 `#0`。
- 管理员返回的 `authorSite`、`authorIp`、`authorUserAgent`、`parentId`、`replyToCommentId` 已在响应中，当前页面只显示邮箱和昵称。增加仅 ADMIN 可见的“详情”展开区：个人网站可安全外链，IP 与 User-Agent 仅用于审核，DEMO 不显示也不应尝试推断。
- 回复仅允许活动且 `PASS` 的评论；通过/隐藏会影响**文章**评论数，留言板不影响；删除/恢复公开文章评论也会同步增减文章评论数。对不可用动作和确认框明确原因与影响。
- 当前 `restore` 只恢复已软删除记录，不等于把 `HIDDEN` 重新公开；隐藏评论应使用“通过”回到 `PASS`。页面动作文案必须区分这两类恢复。

### B-03 分类、标签与友链：防止无法回滚的误操作

- 分类和标签的 slug 创建后不可修改，且格式为小写字母/数字/连字符、最长 64；当前编辑态已禁用并提示，创建态还应在本地校验格式与规范化结果。
- 删除分类或标签会在仍被活动文章引用时被拒绝。删除确认框应提前说明“需先迁移文章”，而不是只在失败后给通用冲突错误。
- 分类与友链的批量排序均要求 1 至 100 个唯一 ID，范围为 0 至 1,000,000，并在服务端事务内全成或全回滚。现有数字输入与保存按钮可复用；增加保存前的批量影响说明即可，不新增拖拽排序。
- 友链 `VISIBLE` 才会出现在公开友链页；`HIDDEN` 不会删除记录。状态切换确认框应写明公开影响。友链名称最长 64，URL/头像仅允许无用户信息的绝对 HTTP/HTTPS URL，介绍最长 255。

### B-04 附件：上传不是“把任意文件放进存储”

- 后端按实际内容识别图片，不以扩展名或前端 `accept` 为准；最大文件为 10 MiB，前端应在选文件后先显示该限制并做同等的本地拒绝，服务端校验仍是最终边界。
- 上传以 SHA-256 去重：相同二进制不会创建新附件；若同一附件在回收站，上传会恢复原记录。完成提示应使用“已上传或复用已有附件”，避免误导用户以为生成了新资源。
- 删除附件是软删除，不会立即删物理对象；活动/回收站切换、恢复入口已存在。封面、Logo、头像等引用在保存时要求活动图片，故删除前需要提醒可能影响后续编辑。

### B-05 站点配置与账号资料：区分全量覆盖、部分更新和公开字段

| 页面 | 后端事实 | 前端应补充 |
| --- | --- | --- |
| 站点配置 | `PUT /api/admin/site-config` 必须携带全部 14 个字段；这是固定 ID=1 的全量覆盖，不是字段级 PATCH | 保存前说明会以当前表单完整覆盖配置；刷新后再编辑，避免用陈旧表单覆盖其他改动 |
| 站点三语 | 后端只要求中文标题；日文/英文标题、摘要、关于页为空时前台逐字段回退中文 | 取消前端对日文/英文标题的额外必填，改为“留空即回退中文”提示 |
| 站点字段格式 | 标题最多 128、副标题 255、关于 Markdown 每语言最多 50,000；Logo/Favicon 只接受绝对 HTTP/HTTPS URL；Spotify ID 仅允许字母数字、`_`、`-`，最长 64 | 使用现有表单校验和字段提示在提交前反馈；不增加新的配置接口 |
| 个人资料 | 后端是 PATCH，允许只提交变更字段；当前前端提交完整表单也能正确清空字段 | 标明资料会公开到前台；对邮箱、URL、昵称和 5000 字简介做本地格式/长度校验，保留服务端最终校验 |
| 修改密码 | 新密码 8–128 字且不能与当前密码相同；成功后服务端撤销该账号所有 refresh token，当前前端已主动退出本机 | 提交前明确“所有设备需要重新登录”，并把“不得与当前密码相同”作为本地错误，而不是笼统请求失败 |

## 6. 不应为了覆盖率而实现的接口

以下 API 封装目前只在自身测试中使用，但列表响应已经包含页面需要的完整字段。新增详情页、弹窗或额外请求没有业务价值，应保留封装和测试，不新增交互：

- `GET /api/admin/categories/{id}`
- `GET /api/admin/tags/{id}`
- `GET /api/admin/friend-links/{id}`
- `GET /api/admin/attachments/{id}`

同理，后端没有支持的文章排序、友链全局筛选、附件按名称/类型查询、统计跨日唯一访客等能力，不得在前端制造“看起来可用”的开关或标签。需要这些能力时单独进行后端契约设计。

## 7. 推荐实施批次与验证

每批只改管理端，并在本地完成对应测试后再进入下一批：

1. **查询正确性（已完成）**：A-01 文章全量筛选、A-02 统计日期区间、A-03 友链无效筛选清理均已在当前分支完成。后续修改筛选 UI 时回归 `query.ts`/hook/组件测试、`pnpm --dir frontend/apps/admin typecheck`、`pnpm --dir frontend/apps/admin test`。
2. **文章和评论语义**：B-01、B-02。重点验证状态切换、时间范围、回复可用性、留言板目标 ID 和 ADMIN/DEMO 的隐私字段。
3. **管理表单语义**：B-03 至 B-05。重点验证 slug、引用保护提示、排序限制、附件去重提示、全量配置字段、语言回退和改密退出。
4. **阶段验证**：管理端 `pnpm --dir frontend/apps/admin build`；不改后端时不以 Maven 全量测试替代前端验证。

## 8. 本轮明确未包含

- 后台视觉重做、布局、主题或组件库替换。
- 新增或修改后端 API、数据库、权限、统计口径、对象存储策略。
- PASSWORD 文章解锁、文章相邻导航、音乐播放器等已在开放问题中独立跟踪的产品增量。
- 任何前端本地筛选/排序对服务端分页结果的伪替代。
