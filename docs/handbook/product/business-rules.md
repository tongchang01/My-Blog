# 当前业务规则

> 状态：当前有效
> 适用范围：MyBlog V2 产品行为
> 最后校准：2026-07-10
> 对应代码：`MyBlog-springboot-v2/src/main/java/`、`frontend/apps/`
> 权威程度：业务规则

## 全局

- 展示语言固定为 `zh / ja / en`；日文或英文内容为空时回退中文。
- 文章正文只保存一份 Markdown，不维护三语正文副本。
- 业务时间按 `Asia/Tokyo` 解释，API 本地日期时间字符串不附加时区后缀。
- 数据库不使用外键；引用存在性、删除影响和恢复条件由应用服务校验。
- 标准业务实体采用软删除，关联表、refresh token、访问明细、日聚合和邮件日志按模型例外处理。

## 身份与权限

- 可登录账号类型为 ADMIN 和 DEMO；GUEST 只表示公开访问语义，不是访客注册体系。
- ADMIN 可以执行后台读写；DEMO 只能读取经过裁剪的数据。
- DEMO 读取文章详情时，只有 PUBLISHED 正文可见；评论管理中的邮箱、IP 和 User-Agent 会被隐藏。
- 登录失败按“IP + 用户名”累计并短期锁定；计数和锁定时间存入账号记录。
- access token 携带用户 ID、账号类型和 token version；refresh token 明文只返回客户端，服务端保存哈希。
- refresh token 每次刷新后轮换；退出、改密或 token version 变化会让旧会话失效。

## 文章

| 状态 | 公开行为 |
| --- | --- |
| DRAFT | 不公开 |
| PUBLISHED | 列表和正文公开 |
| PRIVATE | 不公开 |
| PASSWORD | 列表仅公开锁定元数据，正文和评论拒绝访问 |
| SCHEDULED | 到达发布时间后由定时任务发布 |

- 非 DRAFT 文章必须有中文标题、正文和分类。
- PUBLISHED、PASSWORD 和 SCHEDULED 必须有发布时间。
- PASSWORD 必须保存 BCrypt 哈希，其他状态不得保留文章密码。
- 当前没有 PASSWORD 解锁接口或文章访问 token；详情、评论查询和评论提交返回 `403 + 10003`。
- 只有 PUBLISHED 文章可以进入首页槽位；PINNED 最多 1 篇，FEATURED 最多 2 篇。
- 文章公开路径以 ID 定位，slug 可省略并由详情页规范化；分类和标签 slug 唯一且创建后不可修改。
- 公开评论数只统计未删除且审核状态为 PASS 的文章评论。

## 评论与留言板

- 目标由 `target_type + target_id` 表示；ARTICLE 指向文章，GUESTBOOK 固定使用 ID 0。
- 评论最多两层：回复子评论时仍归属同一个顶层 parent，并以 reply-to 指向实际回复对象。
- 不能跨目标回复，也不能回复待审、隐藏或已删除评论。
- 审核状态为 PASS、PENDING、HIDDEN；只有 PASS 在公开端展示。
- 昵称、邮箱和正文必填，个人站点可选且只允许绝对 HTTP/HTTPS URL。
- 后端保存 Markdown 原文，并只向公开端返回经过解析和清洗的 HTML。
- 命中关键词的评论进入 PENDING；公开提交受频率限制和短期重复内容检查。
- ADMIN 可以审核、隐藏、回复、软删除和恢复；邮件通知失败时记录脱敏日志。

## 分类、标签与系统内容

- 分类为平铺结构，支持三语名称和排序；标签由 ADMIN 维护，支持三语名称。
- 站点配置固定为 ID 1 的单行模型，包含三语站点文案、关于页、Logo、favicon、备案号、Spotify 播放列表 ID 和建站日期。
- 附件上传限制为配置的最大尺寸并校验内容类型；SHA-256 相同的内容复用记录，命中软删除记录时恢复。
- 友链状态只有 VISIBLE 和 HIDDEN；公开端只返回 VISIBLE，在线申请流程不属于当前产品。

## 统计

- 访问事件记录可选文章 ID、语言、来源和不可逆访客哈希，不保存原始访客标识。
- 明细按 JST 日期聚合为 PV/UV；后台仪表盘读取趋势、语言分布和热门文章。
- 公开站点汇总只暴露文章数、分类数、标签数和总访问量。
- 明细、撤销或过期 refresh token、邮件失败日志可由维护任务物理清理。
