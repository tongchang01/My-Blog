# ADR-0020：文章保存封面对齐位置

> 状态：提议中
> 适用范围：V2 文章、管理端文章编辑器和博客端封面展示
> 最后校准：2026-07-18
> 对应代码：`MyBlog-springboot-v2/src/main/java/com/tyb/myblog/v2/content/`、`frontend/apps/admin/src/features/articles/`、`frontend/apps/blog/src/features/articles/`
> 权威程度：ADR

## 背景

文章当前只保存封面附件 ID。博客端文章卡片和详情页背景使用 `cover` 填充容器，并按默认中心位置裁剪；当图片主体不在中心或容器宽高比变化时，主体可能离开可见区域。

相同附件可能被不同文章以不同构图使用，因此该参数属于文章与封面的使用关系，不属于附件本身。

## 决策

- 每篇文章保存 `coverPositionXPercent` 和 `coverPositionYPercent` 两个整数百分比，闭区间为 0–100，默认值均为 50。
- 两个值表示 CSS `object-position` / `background-position` 的对齐位置，不承诺在所有容器宽高比中把同一兴趣点严格居中；管理端必须展示实际 `cover` 效果供作者调整。
- 数据库使用两个非空整数列并以 50 作为默认值。实施时使用迁移目录中下一个可用的 Flyway 版本，不预留或改写既有版本。
- 文章创建和完整更新请求必须显式携带两个字段。没有封面或清除封面时保存 50/50；后台和公开文章列表、首页、详情响应均返回两个字段。
- 管理端首版复用现有封面选择区：显示完整原图，支持点击选点、原生 X/Y 控件、单个实际裁剪预览和重置中心，不增加第三方依赖。
- 博客端文章卡片应用 `object-position`，文章详情页背景应用 `background-position`。默认封面继续使用中心位置。
- 任何文章查询响应仍不得包含 PASSWORD 文章的密码明文或哈希；封面字段扩展不能改变 ADMIN/DEMO 的现有数据裁剪边界。

## 结果

- 旧文章通过数据库默认值保持当前中心裁剪，不需要数据回填或派生图片。
- 调整位置只更新文章数据，不重新处理、复制或删除附件对象。
- 实现必须贯通 Flyway、完整写请求、应用命令、领域模型、持久化、后台与公开读模型、两端类型映射和全部封面渲染点。
- 测试至少覆盖 0、50、100、越界拒绝、创建与完整更新、旧数据默认值、管理端选点/重置、卡片与详情页位置应用。

## 明确不做

- 不生成裁剪图或多尺寸派生图。
- 不接入 AI 主体识别或外部图片处理服务。
- 不引入 Panzoom、Canvas 裁剪器、缩放平移或多套尺寸编辑器。
- 不为旧浏览器增加 `object-position` polyfill。

实现范围与完成条件见 [`ISSUE-018`](../start-here/open-issues.md#issue-018文章封面对齐位置缺失)。
