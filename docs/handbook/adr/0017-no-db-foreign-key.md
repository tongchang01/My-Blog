# ADR-0017：不使用数据库外键约束

> 状态：当前有效
> 适用范围：V2 MySQL schema 与写入服务
> 最后校准：2026-07-10
> 对应代码：`MyBlog-springboot-v2/src/main/resources/db/migration/`、`MyBlog-springboot-v2/src/main/java/`
> 权威程度：ADR

## 背景

项目需要明确的模块边界和可控删除语义。数据库级级联会把业务规则隐藏在 schema 中，并增加数据迁移和恢复的耦合。

## 决策

- Flyway 迁移不创建 `FOREIGN KEY`。
- 关联列使用普通索引支撑查询。
- 创建和更新由应用服务校验引用对象是否存在、可见且状态允许。
- 删除、恢复和排序等操作显式处理被引用关系，不使用数据库级联。
- 复合唯一约束和业务唯一索引仍由数据库执行。

## 结果

引用完整性由用例代码和测试承担。新增关联时必须同时补充校验、索引、删除语义和相应测试。
