# ADR-0013：V2 不兼容 V1 数据结构和接口

> 状态：当前有效
> 适用范围：V2 后端、数据库和前端
> 最后校准：2026-07-10
> 对应代码：`MyBlog-springboot-v2/`、`frontend/apps/`
> 权威程度：ADR

## 背景

为 V1 表结构和接口保留兼容包装会把旧命名、魔法值和历史约束继续带入 V2，并增加长期维护成本。

## 决策

V2 使用独立的数据模型、Flyway 迁移、接口契约和前端实现，不提供 V1 兼容层。V1 源码不在当前主线重复保留，只由只读分支 `archive/v1-master-2026-06-26` 保存。

历史内容迁移属于发布准备工作，不构成 V2 运行时兼容要求。

## 结果

- V2 可以按当前领域语义设计表、枚举和接口。
- V1 只用于历史查阅，不参与当前主线的启动、测试和配置。
- 新功能只在 V2 目录实现，不恢复 V1 兼容代码或目录。

当前 V2 结构见 `../architecture/module-map.md` 和 `../architecture/schema-design.md`。
