# 记录架构决策

> 状态：当前有效
> 适用范围：影响长期架构、安全、数据或技术选型的决策
> 最后校准：2026-07-10
> 对应代码：`docs/handbook/adr/`
> 权威程度：标准流程

以下变化需要 ADR：框架或主要依赖、模块边界、分层方向、认证授权、数据迁移策略、持久化策略和跨模块通信方式。普通缺陷修复、局部重构和既有规则内的实现不需要 ADR。

1. 使用尚未出现的新编号；已删除编号不复用。
2. 按 `../adr/README.md` 模板记录背景、决策、结果、状态、范围、校准日期和对应代码。
3. 说明实际权衡与代价，不写工作日志、教程或实现计划。
4. 同步修改受影响的 architecture、rules、workflows、产品规格和自动化守护。
5. 新决策取代仍有历史价值的旧 ADR 时，把旧记录标为“已取代”并互相链接。
6. 旧 ADR 已完全失效且不再解释当前约束时可以删除，原因由提交记录保留。
7. 验证所有链接、代码路径和状态描述。

ADR 解释“为什么”，当前操作规范仍归属 rules 与 workflows。
