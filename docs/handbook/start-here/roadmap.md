# 路线图

> 状态：当前有效
> 适用范围：MyBlog V2 后续实施顺序
> 最后校准：2026-07-21
> 对应代码：`MyBlog-springboot-v2/`、`frontend/apps/`
> 权威程度：路线图

## 1. 生产运行收口

- 按发布检查清单补齐数据库备份恢复、S3 全链路、可信代理、端口边界和应用回滚的演练记录。
- 观察 `main` 的自动部署与公网 HTTPS 冒烟；处理失败时先保留现场和日志，不假定数据库可随镜像自动回滚。

## 2. 内容与公开体验收口

1. 修复博客核心入口的原生交互语义和标题层级（ISSUE-019）。
2. 按当前主线重新校准已发布源码学习文章和发布填写单（ISSUE-021）。

以上两项均为 P1；按可独立验证和回滚的任务分别实施，不与视觉改版、后台字段或 API 改造混合。

## 3. 产品增量

优先级由实际使用需求决定：

1. 文章封面对齐位置（ISSUE-018）。
2. 文章上一篇/下一篇导航（ISSUE-008）。
3. Spotify Embed（ISSUE-010）与分享/搜索分发能力。

## 4. 触发式架构扩展

- 多人或更高安全要求触发 HttpOnly Cookie 与 CSRF 设计。
- 多实例部署触发共享限流与运行协调。
- 明确搜索收录目标后再实现 sitemap、RSS/Atom、Open Graph 和结构化数据。
- 需要横向扩展时再设计共享限流与运行协调；当前单实例 CD 保持 Docker Compose + GHCR 的可迁移边界。

具体完成条件见 `open-issues.md`。
