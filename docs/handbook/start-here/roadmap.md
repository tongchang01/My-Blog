## 1. 已确认缺陷整改

按一个业务根因一个提交推进，先完成局部回归，阶段末再执行三端完整验证：

1. 文章草稿排除 PASSWORD 口令、隔离用户并接入会话清理（ISSUE-011）。
2. 修正分类/标签复用路由后的查询刷新和旧请求竞争（ISSUE-012）。
3. 修正公开静态页漏记与 canonical slug 重复 PV（ISSUE-013）。
4. 原子化评论审核、删除、恢复和回复的状态与计数变更（ISSUE-014）。
5. 确认根评论失效后的回复展示规则，再统一评论树与计数（ISSUE-015）。
6. 将首页槽位上限改为持久化原子约束（ISSUE-016）。
7. 增加最小运行中 API 契约层（ISSUE-017）。

## 2. 生产运行收口

- 按发布检查清单补齐数据库备份恢复、S3 全链路、可信代理、端口边界和应用回滚的演练记录。
- 观察 `main` 的自动部署与公网 HTTPS 冒烟；处理失败时先保留现场和日志，不假定数据库可随镜像自动回滚。
- 低优先级单独升级 `aws-actions/configure-aws-credentials`，不与业务修复混合。

## 3. 产品增量

优先级由实际使用需求决定：

1. PASSWORD 文章独立解锁授权。
2. Spotify Embed 与分享/搜索分发能力。

## 4. 触发式架构扩展

- 多人或更高安全要求触发 HttpOnly Cookie 与 CSRF 设计。
- 多实例部署触发共享限流与运行协调。
- 明确搜索收录目标后再实现 sitemap、RSS/Atom、Open Graph 和结构化数据。
- 需要横向扩展时再设计共享限流与运行协调；当前单实例 CD 保持 Docker Compose + GHCR 的可迁移边界。

具体完成条件见 `open-issues.md`。
