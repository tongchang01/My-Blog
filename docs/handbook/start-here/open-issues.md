## ISSUE-002：生产部署拓扑和恢复证据缺失

- 优先级：P0，发布阻塞。
- 现状：prod profile、S3、health、CI 和检查清单存在，但服务器、域名、代理、MySQL、证书、备份和回滚尚未确认。
- 完成条件：补齐 `../ops/deployment-direction.md` 的环境事实，完成真实 MySQL 备份恢复、S3、CORS、可信代理、手动发布、冒烟和回滚演练。

## ISSUE-003：PASSWORD 文章无法解锁

- 优先级：P1，产品增量。
- 现状：列表返回锁定元数据；详情、评论查询和评论提交固定返回 `403 + 10003`。系统没有 unlock API、文章访问 token 或前端解锁态。
- 完成条件：单独设计与登录 token 隔离的短期授权，覆盖详情、评论、过期、撤销、前端存储和安全测试，并同步 ADR/API/业务规则。

## ISSUE-004：留言板缺少博客端入口

- 优先级：P2，产品增量。
- 现状：`GET/POST /api/public/guestbook/comments` 已存在，博客路由和页面未接入。
- 完成条件：增加页面、分页、两层回复、提交状态、错误恢复、限流提示和前端测试。

## ISSUE-005：管理端 token 存储风险

- 优先级：P2，触发式安全改造。
- 现状：access/refresh token 存入 `myblog-admin-session` localStorage；XSS 可以读取长期凭证。
- 当前约束：后台不渲染未清洗 HTML，token 不进入 URL、日志或第三方 SDK，refresh 轮换与后端撤销保持启用。
- 重开条件：后台面向多人、引入更多不可信脚本、部署统一为同源 Cookie 架构，或安全等级提高。届时同时设计 HttpOnly、SameSite、CSRF、CORS 和本地开发方案。

## ISSUE-006：单实例限流不能横向扩展

- 优先级：P3，部署触发。
- 现状：登录失败、评论频率/重复提交和访问打点限流使用 Caffeine，各实例状态不共享。
- 重开条件：部署两个以上后端实例。完成方案需提供共享原子计数、过期、容量、降级和测试，不默认绑定某个中间件。

## ISSUE-007：前端构建债务

- 优先级：P3，非发布阻塞。
- 现状：博客使用 Sass legacy API 与 `@import`，主 chunk 约 500 kB；管理端 bundle 较大，Browserslist 与 baseline 数据提示过期。
- 完成条件：分别评估 Sass module 迁移、代码分包、依赖数据更新和性能预算，避免与业务功能提交混合。

## ISSUE-008：博客端非阻塞体验补齐

- 优先级：P3，上线后体验优化。
- 现状：作者卡总字数没有可靠数据源，当前显示 `0`；公开作者资料中的 LinkedIn、Qiita、个人主页和公开邮箱尚未映射到现有 UI；文章正文与评论之间的上一篇、下一篇导航尚未恢复，公开文章详情接口也没有返回相邻文章。
- 当前约束：作者卡已经展示 GitHub、Twitter、知乎和掘金；文章顶部字数和阅读时间由前端根据当前正文计算。这些缺口不阻塞第一版发布。
- 完成条件：确定公开文章总字数的统一统计口径并由可复用数据源提供；补齐需要展示的社交入口；后端按发布时间和 ID 提供相邻文章信息，前端恢复上一篇、下一篇组件。不得为了作者卡在每次访问时下载并扫描全部文章正文。
