# 开放问题

> 状态：当前有效
> 适用范围：MyBlog V2 尚未解决的产品与工程事项
> 最后校准：2026-07-19
> 对应代码：`MyBlog-springboot-v2/`、`frontend/apps/`

本文件只保留仍需行动或满足触发条件后需要重开的事项。已关闭问题由 Git 历史追溯。

## ISSUE-002：生产恢复演练与运行证据未闭环

- 优先级：P1，运行保障。
- 现状：V2 已运行在 AWS EC2；GitHub OIDC SSH CD、GHCR 同 SHA 镜像发布和三条公开 HTTPS 健康检查均已通过。自动冒烟只证明入口可访问，数据库恢复、S3 写入/软删除/恢复、可信代理和应用回滚仍缺少可追溯的生产演练记录。
- 完成条件：按 `../ops/production-runbook.md` 与 `../ops/release-checklist.md` 分别完成并记录数据库备份恢复、S3 全链路、代理与端口边界、应用回滚和一次产品级冒烟；敏感资源标识只写入私有生产台账。

## ISSUE-005：管理端 token 存储风险

- 优先级：P2，触发式安全改造。
- 现状：access/refresh token 存入 `myblog-admin-session` localStorage；XSS 可以读取长期凭证。
- 当前约束：后台不渲染未清洗 HTML，token 不进入 URL、日志或第三方 SDK，refresh 轮换与后端撤销保持启用。
- 重开条件：后台面向多人、引入更多不可信脚本、部署统一为同源 Cookie 架构，或安全等级提高。届时同时设计 HttpOnly、SameSite、CSRF、CORS 和本地开发方案。

## ISSUE-006：单实例限流不能横向扩展

- 优先级：P3，部署触发。
- 现状：登录失败、评论频率/重复提交和访问打点限流使用 Caffeine，各实例状态不共享。
- 重开条件：部署两个以上后端实例。完成方案需提供共享原子计数、过期、容量、降级和测试，不默认绑定某个中间件。

## ISSUE-007：前端构建债务

- 优先级：P3，非发布阻塞。
- 现状：博客已消除 Sass legacy API 与 `@import` 弃用提示，并将 Vue、Markdown 稳定依赖拆出，入口 chunk 由约 748 kB 降至约 360 kB（gzip 约 127 kB）。Mermaid 继续按图表类型加载，最重的动态 chunk 约 691 kB（gzip 约 155 kB），构建阈值按该边界设为 700 kB。管理端 Browserslist 与 Baseline 数据已在 2026-07-18 更新，生产构建不再报告数据过期；管理端 bundle 仍较大。
- 完成条件：为管理端评估代码分包并建立可执行的性能预算，避免与业务功能提交混合；浏览器兼容数据后续作为锁文件维护事项定期更新。

## ISSUE-008：博客端非阻塞体验补齐

- 优先级：P3，上线后体验优化。
- 现状：文章正文与评论之间的上一篇、下一篇导航尚未恢复，公开文章详情接口也没有返回相邻文章。
- 当前约束：作者卡已接入公开作者资料中的 GitHub、Twitter、知乎、掘金、LinkedIn、Qiita、个人主页和公开邮箱；文章顶部字数和阅读时间由前端根据当前正文计算。这些缺口不阻塞第一版发布。
- 完成条件：后端按发布时间和 ID 提供相邻文章信息，前端恢复上一篇、下一篇组件。

## ISSUE-009：V2 前后端能力对齐审查

- 优先级：P2，上线稳定后启动。
- 背景：站点配置、作者资料、友链、留言板和管理端密码修改已经完成对齐，但此类“后端已有接口、前端没有可用入口”的缺口不能靠人工记忆持续发现。
- 已完成审查：2026-07-15 已按实际管理端路由、调用点和后端服务复核[管理端后端能力消费审查](../frontend/admin/backend-consumption-audit.md)。文章全量筛选、统计日期范围和友链无效筛选均已在当前分支完成前端对齐；其余详情接口是否保留已有明确结论。下一轮聚焦表单业务语义、操作反馈和视觉改造前的上游模板清理。
- 范围：以 `docs/handbook/api/` 和实际代码为事实来源，建立接口、前端调用、页面入口、权限和测试的映射。后端已有但前端缺失的能力优先评估并补齐；前端已有但后端缺失的需求必须先确认产品价值、安全边界和验收标准，再决定实现或删除前端预期。
- 约束：不以一次大重写为目标；按业务模块拆分为可独立测试、独立发布和独立回滚的小任务。认证、权限、密码、附件和数据写入等安全敏感能力优先。
- 完成条件：完成可追溯的能力清单；每个不一致项都有“补前端、补后端、删除预期或明确暂缓”的结论；已选项补齐对应 API 契约、前后端测试和发布验收。

本轮博客端对齐已完成站点配置、作者资料、友链错误恢复、留言板入口和 PASSWORD 文章解锁；相邻文章导航仍按上述独立事项处理。

## ISSUE-010：博客音乐体验

- 优先级：P2，体验增量，非发布阻塞。
- 方向：不恢复旧版权/主题署名页脚，也不把播放器固定在页面末尾。导航栏提供音乐入口；用户主动打开后显示播放面板；仅在有活动播放时显示右下角迷你控制条，以便跨页面暂停、继续或展开。
- 当前约束：播放器的曲目来源、版权/授权、是否需要登录和播放列表管理尚未设计；未明确前不得接入第三方音乐服务或上传媒体文件。默认状态不显示常驻悬浮条，避免干扰文章阅读。
- 完成条件：确定媒体来源与授权边界；完成导航入口、播放面板、跨路由播放状态、活动迷你控制条、移动端布局、键盘可操作性及播放失败状态；通过前端测试和主要页面回归。

## ISSUE-018：文章封面对齐位置缺失

- 优先级：P2，内容展示体验。
- 现状：文章只保存封面附件 ID，博客端卡片和文章详情页背景均按默认中心位置裁剪；主体偏离中心时可能离开可见区域。
- 已确认方向：按 [`ADR-0020`](../adr/0020-article-cover-position.md) 在文章上保存 0–100 的 X/Y 整数百分比，默认 50/50；管理端使用无新增依赖的选点与实际裁剪预览，博客端应用原生 CSS 对齐位置。
- 范围约束：不生成派生图，不引入 Panzoom、Canvas 裁剪器、AI 检测或外部图片服务；不得让 PASSWORD 密码明文或哈希进入任何查询响应。
- 完成条件：使用下一个可用 Flyway 迁移贯通文章完整写入和后台/公开读取；管理端支持完整原图选点、X/Y 控件、单个裁剪预览和重置中心；普通卡片、首页横向卡片与详情页背景均应用位置；后端、管理端、博客端和运行中 API 契约覆盖默认值、边界、越界与完整 PUT。

## ISSUE-019：博客核心入口缺少原生交互语义

- 优先级：P1，公开阅读可达性。
- 现状：桌面鼠标可以完成导航、搜索和文章打开，但顶部导航、Logo、语言/搜索/主题控制及文章卡片主要由带点击事件的 `div`、`span` 或 `href="javascript:void(0)"` 实现。键盘 Tab 无法到达核心入口，文章卡片也不具备复制链接、在新标签页打开和屏幕阅读器识别所需的原生链接语义。首页文章卡片重复使用 `h1`，文章详情存在两个 `h1`，留言板没有页面级 `h1`。
- 范围：复用 Vue Router 与现有样式，把站内跳转收口为 `RouterLink`/真实 `a`，动作收口为 `button`；不得以只补 `tabindex` 或点击键盘事件的方式继续保留伪交互元素。同步修正页面标题层级和 404 恢复入口，不在本事项中改视觉风格。
- 完成条件：鼠标、键盘 Enter/Space、Tab 顺序和原生链接操作均可使用；控件具有可识别名称和清晰焦点；每个公开页面只有一个符合内容层级的 `h1`；首页、筛选结果、搜索、文章详情和 404 的组件测试及真实浏览器回归通过。

## ISSUE-020：评论与留言板界面未贯通三语

- 优先级：P1，公开多语言一致性。
- 现状：中文、英文和日文路由及页面标题可以正常切换，但文章评论区与留言板的输入提示、提交状态、回复、空状态和分页文案仍硬编码为中文。请求携带当前 locale 不能替代界面文案国际化。
- 范围：复用现有 `vue-i18n` 资源和 locale 状态，不新增接口、字段或独立国际化框架；同时覆盖文章评论与 guestbook 共用组件的全部用户可见状态。
- 完成条件：`zh`、`en`、`ja` 下表单、回复、加载、空数据、提交中、成功/失败提示和分页文案与当前路由一致；同页切换语言立即更新；三语组件测试与文章、留言板真实页面回归通过。
