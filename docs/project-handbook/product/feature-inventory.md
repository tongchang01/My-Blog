# V1 功能清单（考古结果）

> 本文档：把 V1 三端的所有功能盘点出来，逐条给出 **AI 推荐 + 你最终决定**，作为 V2 重设计的输入。
> 考古时间：2026-06
> 数据源：`MyBlog-springboot/`（V1 后端，20 个 Controller）、`MyBlog-vue/MyBlog-blog/`（前台 Vue 3）、`MyBlog-vue/MyBlog-admin/`（后台 Vue 2）

## 使用方式

每行有两列：

- **AI 推荐**：我（AI）基于"个人博客 + 极小数据量 + 仅你自己运营"视角的建议
- **你的决定**：你接受 / 推翻 / 改写。填 **留 / 改 / 删 / ?** 之一即可

值的含义：

- **留** — V2 保留，行为基本不变
- **改** — V2 保留但要重设计（备注里写改成什么样）
- **删** — V2 不要
- **?** — 待你想清楚

如果你完全认同 AI 推荐，可以在【你的决定】栏直接写"同上"。
填完后我们一起根据这份清单出 `use-cases.md` / `data-model.md` / `arch/schema-design.md`。

## AI 推荐总思路

- **核心三件套保留**：内容（文章/分类/标签/归档/搜索）、互动（评论/留言/友链）、身份（登录/管理员/网站配置）
- **企业级冗余砍掉**：Quartz、动态菜单、动态资源 RBAC、操作日志入库、异常日志入库、在线用户管理
- **维护成本高的内容载体砍掉或外接**：自建相册（用图床）、自建音乐（不要）、说说（不要）
- **借机补现代博客标配**：暗黑模式、RSS、Sitemap、SEO、阅读时长、字数统计、TOC 锚点、代码块复制、文章版权协议、评论邮件通知

## 五、明显的"砍掉候选"汇总（我的推荐）

按"个人博客 + 极小数据量 + 仅自己运营"的视角，这些功能维护性价比低：

| #   | 功能                       | 砍的理由                                    |
| --- | ------------------------ | --------------------------------------- |
| 1   | 定时任务 Quartz + 后台管理       | 个人博客无业务定时任务，技术定时任务用 `@Scheduled` 写死即可   |
| 2   | 操作日志入库 + 后台查看            | 走日志文件，文件够看，不入库                          |
| 3   | 异常日志入库 + 后台查看            | 同上，走日志文件                                |
| 4   | 复杂 RBAC（菜单 + 资源 + 角色多对多） | 两角色 USER/ADMIN + 静态菜单足够                 |
| 5   | 后台菜单动态管理                 | 静态路由更简单也更稳                              |
| 6   | 音乐播放器 + 后台管理             | 嵌一个第三方播放器或不要                            |
| 7   | 相册系统（自建图床）               | 用图床（七牛/腾讯云 COS / GitHub）；正文图片上传一个简易上传即可 |
| 8   | 说说功能                     | 你不写就删                                   |
| 9   | QQ OAuth                 | 看你是否需要让访客评论用 QQ 登录                      |
| 10  | 文章导入/导出                  | 个人写文章很少批量导入                             |
| 11  | 在线用户管理 / 强制下线            | 需要 Redis；个人博客你也不会真去踢谁                   |
| 12  | 仪表盘 echarts 全家桶          | 简化为"总文章数 / 总评论数"几张数字卡足够                 |
| 13  | 国际化 vue-i18n             | 你不真的多语言就不要                              |
| 14  | 评论 type=5 的"说说评论"        | 跟说说一起砍                                  |

**保留没争议的核心**：文章 / 分类 / 标签 / 评论（针对文章）/ 友链 / 关于我 / 留言板 / 用户基本登录 / 网站配置 / 文章搜索 / 文章归档。

## 七、AI 给的"V2 最终建议形态"草图

如果你完全采纳我的推荐，V2 会是这样：

- **后端**：1 个 SpringBoot 服务 + 6 模块（已规划）+ ~10 张表（article/article_tag/category/tag/comment/friend_link/user_auth/user_info/site_config/page_view_daily）
- **前台**：Vue 3 + Element Plus + Pinia，8 个页面（首页/文章详情/分类页/标签页/归档/留言/友链/关于）+ 暗黑模式 + RSS + Sitemap + 评论邮件通知
- **后台**：Vue 3 + Element Plus + Pinia（统一前台），8 个页面（登录/仪表盘/文章/分类/标签/评论/友链/设置）+ Vditor 编辑器 + 图床抽象
- **角色**：枚举 ADMIN/USER（USER 实际不存在数据库，只是占位），无注册，无 OAuth
- **统计**：自研只做"日聚合 PV/UV"；详细统计接 Umami
- **运维**：自动备份脚本 + 简单限流 + 评论 XSS 清洗 + 上传白名单
