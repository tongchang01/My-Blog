# V2 功能范围决策清单

> 本文档回答一个问题：V2 到底保留、重做、删除、新增哪些功能？
>
> 适用范围：产品规格、接口契约、数据库设计、前后台页面设计。
>
> 当前状态：待逐项确认。本文整理 V1 考古结果和 AI 建议，不代表最终决定。

## 0. 使用规则

`你的决定` 只建议填写以下值：

| 值 | 含义 |
| --- | --- |
| 留 | V2 保留，行为基本不变 |
| 改 | V2 保留，但要重做交互、数据结构、接口或实现方式 |
| 删 | V2 不再提供 |
| 新增 | V1 没有，V2 增加 |
| 待定 | 暂不拍板，需要后续讨论 |

建议先只处理 `1. 优先决策区`。这一部分定下来以后，再反推 `use-cases.md`、`data-model.md`、接口设计和前后台页面。

## 4. 明确建议删除的 V1 功能

| 功能 | 删除原因 | 替代方式 |
| --- | --- | --- |
| 动态菜单 | 单人博客不需要后台配置菜单 | 前端静态路由 |
| 动态资源权限 | 接口级 RBAC 成本高 | 后端固定权限注解或简单角色 |
| 角色管理 | 只有管理员时价值低 | 用户表保留角色字段 |
| 在线用户管理 | 维护价值低 | 登录态自然过期 |
| Quartz 后台任务 | 业务上没有强需求 | 必要任务用固定 `@Scheduled` |
| 任务日志入库 | 表膨胀且难维护 | 应用日志 |
| 操作日志入库 | 单人后台价值低 | 应用日志 |
| 异常日志入库 | 观测能力弱且表会膨胀 | 应用日志、Sentry、Loki |
| 说说 | 不是博客核心 | 外部社交平台或暂不做 |
| 相册 | 图床和相册维护成本高 | 外部图床、对象存储 |
| 音乐播放器 | 非核心且版权/维护成本高 | 不做 |
| QQ 登录 | 访客体系不完整时价值低 | 评论昵称 + 邮箱 |

## 7. 推荐的 V2 目标形态
