# MyBlog：フルスタックな個人ブログシステム

ブログは、記事を書く場所であると同時に、継続的に育てていける個人プロダクトでもあると思っています。

`MyBlog` は、私自身が実際に使いながら改善を続けているブログシステムです。記事投稿やコンテンツ管理だけでなく、コメント、アルバム、つぶやき、定期実行ジョブ、権限管理、音楽プレイヤーなども含めて、より完成度の高いサイトを目指しています。

---

## 1. プロジェクトの位置づけ

このプロジェクトは次の 3 つで構成されています。

- 公開用ブログフロント
- 管理画面
- バックエンドサービス

単なるテンプレートではなく、長期的に拡張・運用できるコンテンツプラットフォームとして設計しています。

---

## 2. 技術構成

### フロントエンド

フロントは 2 つのアプリに分かれています。

- `MyBlog-blog`：`Vue 3 + TypeScript + Pinia + Element Plus`
- `MyBlog-admin`：`Vue 2 + Vuex + Element UI`

公開側は閲覧体験と表現を重視し、管理画面は運用と管理効率を重視しています。

### バックエンド

バックエンドでは主に次を使っています。

- Spring Boot
- Spring Security
- MyBatis-Plus
- MySQL
- Redis
- RabbitMQ
- Quartz
- Knife4j

さらに、

- AWS S3
- AWS SES

も連携しています。

---

## 3. 機能モジュール

### 公開ブログ

公開側には次の機能があります。

- ホームページ
- 記事詳細
- カテゴリ、タグ、アーカイブ
- 友達リンク
- アルバム
- つぶやき
- コメントシステム
- 音楽プレイヤー

### 管理画面

管理画面では以下をサポートしています。

- 記事管理
- カテゴリ管理
- タグ管理
- コメント審査
- つぶやき管理
- アルバム管理
- 友達リンク管理
- メニュー管理
- ロール管理
- リソース権限管理
- ユーザー管理
- オンラインユーザー管理
- 操作ログ / 例外ログ
- 定期実行ジョブ管理
- サイト設定
- 音楽管理

この管理機能群によって、単なるブログではなく、運用可能なコンテンツシステムに近づいています。

---

## 4. 追加したカスタム機能

### 音楽プレイヤー

最近追加した機能の一つが音楽プレイヤーです。

- フロントでの APlayer 統合
- 管理画面での音楽設定と曲管理
- バックエンドの音楽 API
- データベースへの音楽テーブルと権限リソースの追加

UI の挙動も調整しており、

- 右下固定モードは削除
- ドック型のフローティング表示のみ残す
- デスクトップでは位置を上に移動して、右下 UI と重ならないようにする

という形にしています。

### サイト設定

サイト設定では、単なるテキスト入力だけでなく、

- サイト名
- 作者情報
- アバターやロゴ
- お知らせ
- SNS リンク
- コメント審査
- メール通知
- 投げ銭の有効化
- 音楽プレイヤー設定

など、多くの表示要素を管理画面から調整できるようにしています。

### 定期実行とログ

さらに、

- Quartz による定期実行
- ジョブログ
- 操作ログ
- 例外ログ

も組み込んでいます。

これによって、継続運用を前提にした構成になっています。

---

## 5. 3 分割構成にした理由

記事を書くことだけが目的なら、単一アプリでも十分です。

それでも 3 つに分けたのは、

- 公開側と管理側で責務が大きく異なる
- 将来の拡張がしやすい
- 実際の業務システムに近い構成を意識した

からです。

---

## 6. このプロジェクトで表現できること

このプロジェクトは、作品として見たときに次のような点を表現できます。

- フルスタック構成を自分で組み立てられること
- 表示層と管理層を分離して設計できること
- ブログからコンテンツプラットフォームへ拡張していけること
- 権限、ログ、定期実行、設定管理など、運用寄りの課題にも対応できること
- 実際の利用を通じて継続的に改善していること
