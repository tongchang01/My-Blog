# MyBlog V2：保守可能な 3 アプリ構成の個人ブログ

MyBlog V2 は、公開ブログ、管理画面、Spring Boot API で構成されるフルスタックの個人ブログシステムです。重視しているのはミドルウェアの数ではなく、明確な業務境界、実行可能なアーキテクチャ規則、コンテンツ運用の一連の流れ、検証可能なセキュリティ制約です。

## プロダクト機能

公開ブログは中国語・日本語・英語の UI に対応し、ホーム編集、記事詳細、カテゴリ、タグ、アーカイブ、検索、About、リンク、メッセージボード、記事コメント、著者プロフィール、アクセス統計を提供します。記事 URL は数値 ID を安定した識別子とし、任意の可読 slug を付加します。

管理画面はログインセッション、統計ダッシュボード、記事公開・予約投稿、固定・注目枠、カテゴリ・タグ、コメント審査、リンク、添付ファイル、サイト設定、著者プロフィールを扱います。ADMIN は読み書き可能、DEMO は読み取り専用で、機密項目はサーバー側で除外されます。

## エンジニアリング設計

バックエンドは Java 17、Spring Boot 3.5、Spring Security、MyBatis-Plus、Flyway、MySQL 8 を使用します。identity、content、comment、system、stats の 5 業務モジュールは 4 層構造を持ち、モジュール間連携は application 契約を経由します。ArchUnit が依存違反を自動検出します。

認証は短期 JWT access token、DB refresh token、並行実行に対応したローテーション、token version による失効を組み合わせます。コメント Markdown はサーバーで変換・サニタイズされます。添付ファイルはローカル/S3 に対応し、アクセス統計は不可逆な訪問者ハッシュと日次集計を使用します。

2 つのフロントエンドは Vue 3、TypeScript、Pinia、Vite、Vitest を使用します。ブログは Aurora のビジュアル基盤から発展し、管理画面は pure-admin-thin を UI 基盤としています。データと業務操作は V2 REST API に接続済みです。

## 現在の境界

PASSWORD 記事はロック情報のみ公開し、解除フローは未実装です。完全な SEO/feed、Spotify Embed、複数インスタンス協調は、利用要件が明確になった時点で着手する拡張です。

本番環境は AWS EC2 で稼働しています。Docker Compose が MySQL、API、Caddy を実行し、添付ファイルは S3 に保存します。GitHub Actions は GHCR イメージをビルドし、GitHub OIDC と制限付き SSH で同一コミット SHA を自動デプロイします。公開 HTTPS health check はデプロイ後のスモークテストに含まれますが、DB 復旧、S3 全経路、ロールバック演習は継続して検証します。

## このプロジェクトが示すもの

- 状態、権限、データモデルから設計したフルスタック業務フロー。
- モジュール境界、トランザクション、テスト、マイグレーションによる長期的一貫性。
- 公開体験、管理運用、セキュリティ、監査、デプロイ準備を統合した設計。
- 実装済みの事実、既知リスク、将来拡張の明確な区別。
