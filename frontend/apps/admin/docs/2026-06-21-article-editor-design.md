# 后台文章新增与编辑设计

## 目标

在现有文章列表基础上补齐 ADMIN 的文章新增、详情加载和完整编辑能力，使后台写入的数据可以立即出现在文章列表与博客前台。DEMO 保持只读。

## 方案选择

采用独立路由编辑页：`/articles/new` 用于新增，`/articles/:id/edit` 用于编辑。相比列表内编辑，独立页面能容纳三语字段、Markdown 正文和发布设置；相比弹窗，它不会因表单较长造成滚动和状态管理混乱。暂不引入富文本或 Markdown 编辑器依赖。

## 页面结构

- 顶部操作栏：返回列表、保存文章。
- 基础内容卡片：中文标题与摘要为主要字段，日文、英文为可选占位字段。
- 正文卡片：使用多行文本域编辑 Markdown 原文。
- 发布设置卡片：slug、状态、分类、标签、发布时间和 PASSWORD 访问密码。
- 所有非草稿状态必须选择分类；状态为 `SCHEDULED` 时必须填写发布时间；创建 PASSWORD 文章时必须填写密码；编辑 PASSWORD 文章时留空代表保留原密码。
- 新建文章的封面附件字段提交 `null`；编辑文章保留当前封面附件 ID，本批不实现选择或上传。

## 权限与数据流

- 列表继续允许 ADMIN、DEMO 读取。
- 新建和编辑路由仅允许 ADMIN。
- 页面初始化并行读取分类、标签；编辑模式额外读取文章详情。
- 保存时提交后端要求的全部字段；空白可选文本统一转换为 `null`，标签 ID 去重。
- 创建或更新成功后返回文章列表；失败时保留表单并显示错误提示。

## 测试范围

- API：详情、创建、更新的路径、方法和请求体。
- 表单领域逻辑：默认值、详情映射、空值规范化、非草稿分类、定时发布和密码校验。
- 路由与列表：新建/编辑入口、ADMIN 权限。
- 编辑页：初始化、保存成功、校验失败与请求失败。
- 阶段验证：Admin 单测、类型检查、生产构建和真实 MySQL 浏览器联调。

## 暂不实现

- 封面上传和附件选择。
- Markdown 实时预览、自动保存和离开页面确认。
- 分类、标签本身的增删改。
- 回收站、批量操作和站点资料配置。
