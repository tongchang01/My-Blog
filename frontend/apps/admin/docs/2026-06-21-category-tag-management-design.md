# 后台分类与标签管理设计

## 目标

在现有文章管理闭环后补齐分类与标签维护能力，使 ADMIN 无需直接操作数据库即可创建、修改、排序和删除文章引用的内容字典。DEMO 保持只读，后端继续作为权限、唯一性和引用保护的最终边界。

## 页面与路由

沿用文章列表的“独立筛选卡片 + 结果表格卡片”结构，并在现有内容管理菜单组中增加两个独立页面：

- `/categories/list`：分类管理。
- `/tags/list`：标签管理。
- 文章列表、分类管理和标签管理作为同一菜单组的三个可见子项；文章新增和编辑路由继续隐藏。
- 两个列表路由允许 ADMIN、DEMO 读取；新增、编辑、排序和删除控件只对 ADMIN 渲染。

后端分类和标签接口返回完整活动列表且没有分页、筛选参数，因此筛选在前端本地完成。筛选项只有一个关键词，同时匹配三语名称和 slug；不伪造后端分页。

## 分类管理

分类表格展示当前界面语言名称、其他已填写语言徽标、slug、排序值和更新时间。ADMIN 可执行：

- 新增：弹窗填写中文名称、可选日文/英文名称、slug 和排序值。
- 编辑：复用同一弹窗，直接使用当前行完整数据初始化，不额外请求详情。
- 排序：表格内编辑排序值；页面只收集已变更项，通过 `PUT /api/admin/categories/sort-orders` 一次提交。保存成功后重新加载列表。
- 删除：二次确认后调用 `DELETE /api/admin/categories/{id}`。

排序值限定为 `0..1000000`。没有脏数据时禁用“保存排序”，避免空批次请求。

## 标签管理

标签表格展示当前界面语言名称、其他已填写语言徽标、slug 和更新时间。新增、编辑使用弹窗，字段为中文名称、可选日文/英文名称和 slug；删除采用二次确认。标签没有排序能力，界面不提供排序占位。

## 表单与数据规范

- 中文名称和 slug 必填；日文、英文名称去除首尾空白后，空值统一提交 `null`。
- slug 在前端只做必填校验，最终格式规范化、永久唯一与并发冲突由后端保证，避免前后端规则漂移。
- Snowflake ID 始终按字符串传递；分类排序请求中的 ID 也保持字符串，由 JSON 数字字符串交给 Spring 转换为 `long`，避免浏览器精度损失。
- 弹窗保存期间禁用重复提交；成功后关闭弹窗并刷新列表，失败时保留用户输入。

## 状态与错误处理

分类和标签各自使用独立的状态控制器，管理加载、关键词、原始列表、筛选结果、弹窗状态和写请求状态。页面覆盖 loading、empty、error 和 retry。

- `90001`：显示对应表单校验或通用请求错误。
- `90004` 保存冲突：显示本地化的“名称或 slug 已被占用”。
- `90004` 删除冲突：显示本地化的“仍被活动文章引用，无法删除”。
- `10003`：显示无写权限；正常情况下 DEMO 看不到写控件。
- 网络和未知错误：保留当前页面状态并提供重试。

本阶段不依赖后端中文 `msg` 生成用户文案，按错误码和当前操作映射三语提示，避免界面语言混杂。

## 模块边界

```text
src/api/taxonomy.ts                         分类与标签读写请求
src/features/taxonomy/model.ts             DTO、表单和排序请求模型
src/features/taxonomy/form.ts              默认值、映射、校验和请求体规范化
src/features/taxonomy/categories/          分类状态控制器与页面
src/features/taxonomy/tags/                标签状态控制器与页面
src/router/modules/articles.ts             内容管理菜单与路由
locales/{zh-CN,ja,en}.yaml                 三语文案
docs/                                      设计、计划和验收记录
```

分类和标签共享数据类型与纯表单函数，但页面状态控制器分开实现。这样可以复用稳定规则，同时避免把分类排序逻辑塞进标签模块。

## 测试与验收

- API 测试：分类/标签列表、创建、更新、删除，以及分类批量排序的路径、方法和请求体。
- 表单测试：默认值、编辑映射、空值转 `null`、必填字段和排序范围。
- 状态测试：本地筛选、加载失败重试、写入后刷新、脏排序收集与写冲突。
- 页面测试：ADMIN 写控件、DEMO 只读、弹窗新增/编辑、删除确认和错误提示。
- 路由测试：三个可见内容管理子项与 ADMIN/DEMO 访问边界。
- 阶段验证：Admin 全量测试、typecheck、生产构建；使用本地 MySQL 通过浏览器分别新增、编辑和删除一个临时分类与标签，验证数据后彻底清理。

## 暂不实现

- 分类层级、批量删除、拖拽排序和标签合并。
- 在文章编辑弹窗内直接创建分类或标签。
- 服务端分页、服务端关键词筛选和删除记录回收站。
- 修改后端接口、数据库结构或现有五篇文章数据。
